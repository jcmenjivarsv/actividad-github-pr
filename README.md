# Actividad GitHub: Ramas + Pull Request (PR)

## Objetivo
Practicar el flujo básico en GitHub:
1) Crear una rama
2) Hacer un cambio
3) Subirlo (push)
4) Abrir un Pull Request
5) Corregir si hay observaciones

---

## Instrucciones
1. Haz un **fork** de este repositorio a tu cuenta.
2. Clónalo en tu computadora.
3. Crea una **rama** con tu nombre (ej: `add-jose-valle`).
4. Agrega tu archivo dentro de `submissions/`.
5. Haz commit, push y abre un **Pull Request**.

---

## Reglas
- No trabajes en `main`.
- Solo se acepta entrega por Pull Request.
- El archivo debe estar en `submissions/` y seguir el formato indicado.

---

## Formato del archivo (obligatorio)
Crea un archivo en: `submissions/tu-nombre-tu-apellido.md`

Ejemplo: `submissions/jose-valle.md`

Contenido mínimo:
- Nombre completo
- Usuario de GitHub
- Fecha
- Un comentario corto (1 línea)

Ejemplo:
```md
# Jose Valle
- GitHub: @josevalle
- Fecha: 2026-01-03
- Comentario: Aprendí a crear ramas y hacer PR.
